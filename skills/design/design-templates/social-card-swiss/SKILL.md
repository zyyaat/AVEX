# Social Card Swiss Template

Use this template for structured social cards that should feel precise, systematic, technical, or method-driven.

## Use When

- The content is a framework, checklist, comparison, tool explanation, AI workflow, product update, KPI summary, or screenshot explainer.
- The design should use grid, numbering, labels, metrics, or modular blocks.
- The user wants a clean modern card that feels designed but not decorative.

## Avoid When

- The content is emotional, literary, image-led, or slow editorial.
- The user wants a soft lifestyle/pastel look.
- A plain poster or deck slide is a better fit than a social card.

## Required Reading

Read this `SKILL.md` first. Do **not** read `pattern.html` by default.

Read `pattern.html` only if the selected card needs layout relationships that are hard to infer from this file. Extract only grid rhythm, component relationships, responsive/card behavior, and section hierarchy.

## Design Strategy

Choose a card role before choosing visual detail:

- system cover
- numbered method
- comparison / before-after
- KPI tower
- horizontal bar / mini chart
- screenshot explainer
- evidence wall
- closing checklist

## Layout Bias

Prefer:

- strict grid and alignment
- numbering systems and small metadata labels
- clear typographic hierarchy
- compact but readable card sections
- data blocks, comparison rows, and interface frames

Avoid:

- random stickers, emoji icons, and decorative blobs
- unclear focal hierarchy
- screenshots too small to read
- generic SaaS feature-card layouts

## Platform / Size

Use `canvas-and-device.md` for platform dimensions and export ratios. This template supplies structure, not fixed size.
